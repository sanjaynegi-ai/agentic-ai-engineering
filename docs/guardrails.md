# Guardrails, grounding, and boundaries

A **guardrail** is a rule, validation, or runtime check that constrains inputs, outputs, tool use, cost, safety, or scope. **Grounding** requires an answer to rely on supplied data or trusted sources. **Boundaries** define what the system may do. **Validation** enforces data contracts. **Policy** states organizational rules. System prompts can reinforce these controls but cannot replace runtime enforcement.
